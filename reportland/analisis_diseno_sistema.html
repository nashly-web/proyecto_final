1) Propósito y base del documento UI → Diseño

Esta documentación describe el análisis y diseño del sistema SOS EmergeLens. Todo el contenido está construido en base a la información de la página: pantallas (UI), flujos principales y endpoints consumidos por el frontend.

Si algo cambia en la UI o en la API, este documento debe actualizarse para mantenerse alineado con el sistema real.

2) Resumen del sistema Emergencias

Elemento Descripción
Frontend React + Vite (SOS, emergencia activa, llamada LENS, admin, historial).
Backend Flask API (auth, emergency, lens_call, audit, reports, geofence, etc.).
Datos Odoo (modelos de perfil/emergencia/notificación).
Notificación Correo (SMTP) + panel de notificaciones in-app.
Voz LENS (voz + transcripción visible + turn-taking).

3) Actores y casos de uso

Análisis
Actor Descripción Pantallas relacionadas
Usuario Activa SOS, comparte ubicación/evidencia, llama a LENS, gestiona perfil y contactos. Home, EmergencyActive, CallSimulator, Profile, Contacts
Contacto Recibe correo con ubicación/datos del incidente. Externo (correo)
Admin/Operador Monitorea alertas, asigna unidad, actualiza estados, revisa auditoría/reportes. Dashboard, AdminAlerts, Notifications, History

Casos de uso principales (derivados de la UI)

UC

UC-01 Activar SOS

Seleccionar tipo (medical/security/fire/accident) y presionar SOS. Registrar alerta, compartir ubicación y enviar correo.

UC-02 Seguimiento

Actualización de ubicación y batería en tiempo real durante la emergencia.

UC-03 Evidencia + LENS

Foto automática + audio opcional; llamada LENS con transcripción y respuesta contextual.

4) Requisitos (RF / RNF)

Diseño
ID Requisito funcional Módulo (UI/API)
RF-01 Activar SOS por tipo y registrar alerta. /api/emergency/*
RF-02 Compartir ubicación en vivo. /api/emergency/location
RF-03 Enviar correo SOS con mapa y datos del incidente. /api/emergency/email
RF-04 Capturar/enviar evidencia (foto/audio), con reenvío. /api/emergency/evidence
RF-05 Asignar y mostrar unidad (ambulancia/policía/bomberos/rescate). /api/emergency/unit/*
RF-06 Llamada con LENS: voz + transcripción visible + respuesta contextual. /api/lens/message

RNF (no funcionales)

RNF
  • RNF-01 Rapidez: flujo diseñado para operar en segundos.
  • RNF-02 Disponibilidad: fallbacks si integraciones externas fallan.
  • RNF-03 Seguridad: sesiones, control de acceso, PIN, auditoría.
  • RNF-04 Usabilidad: UI clara y mensajes cortos en crisis.
  • RNF-05 Privacidad: evidencia y datos solo para actores autorizados.

7) API (endpoints clave)

REST
Grupo Endpoint Método Propósito
Auth /api/auth/login, /api/auth/register, /api/auth/logout, /api/auth/me POST/GET Sesión y datos del usuario.
Emergencia /api/emergency/email POST Envía correo SOS y registra alerta.
Emergencia /api/emergency/location, /api/emergency/battery POST Seguimiento en vivo (ubicación y batería).
Evidencia /api/emergency/evidence POST/GET Envía/consulta evidencia (foto/audio).
LENS /api/lens/message POST Respuesta contextual basada en lo dicho por el usuario.
Geofence /api/geofence/* GET/POST/PATCH/DELETE Zonas seguras y eventos.
Audit/Reports /api/audit/*, /api/reports/* GET/POST Trazabilidad, exportes y estadísticas.
